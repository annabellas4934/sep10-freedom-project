## Content:
We have finally started working on our freedom project, using everything we learned within the past year to make a website. We were given a week to finish our Minimal Viable Product. When I started I struggled implementing Bulma with Bootstrap because I had to work with them separately, but not together, and they had similar classes that conflicted with each other. At first, I tried to use only Bulma, but then I realized that there were components in Bootstrap that I needed for my project to look like how I wanted for my wireframe. I looked it up and according to the [Github community](https://github.com/jgthms/bulma/issues/1865) it works but with different classes.according to [Bulma’s site](https://bulma.io/alternative-to-bootstrap/) it can serve as a good alternative. I did use a lot of Bulma’s content for my website, but bootstrap was much simpler for the grid system. Eventually I came up with an MVP, though I’m not satisfied with it, it serves its purpose.

My favorite thing I have in my project is the pagination at the bottom of every page. I got this from my tool, Bulma. It is usefull for when finishing reading the content, you can switch pages without using a navbar, similar to a book.
<img src="bulmablog5.png">
This is a screenshot of how the code looks like on piccode, the platform where I did most of my tinkering. On piccode, you could see how the code affects the website diresctly next to the codespace, so when figuring out if the buttons would link to the correct pages, or whetehr of not the buttons were active or not, this was most useful. When `class="pagination-link is-current"` it means that you are on the page the link is on, and it will not allow you to click it again because there is no point in going to a page you are already on. `class="pagination-next is-disabled"` was used at the end because the is no next page so the button is disabled and turns grey to signal you can't use it. There are so many mini details in the pagination which makes it really useful.
